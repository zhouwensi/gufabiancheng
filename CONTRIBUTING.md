# 贡献指南

感谢你愿意参与古法编程的传承。

## 参与方式

### 1. 提交濒危技艺

你发现某个游戏编程技艺正在消失？请通过 Issue 告诉我们：

- 技艺名称
- 简要描述
- 你认为的濒危等级（极危 / 濒危 / 易危 / 近危 / 已灭绝）
- （可选）你与这项技艺的故事

### 2. 分享古法故事

你有过不借助 AI、纯手工开发游戏的经历？我们想听你讲：

- 你做了什么
- 用了什么技术
- 过程中最痛苦和最快乐的部分
- （可选）项目链接

### 3. 推荐殿堂作品

你认为哪些游戏值得进入"古法游戏殿堂"？标准：

- 核心代码由人类手写
- 开发过程体现了极高的手工技艺
- 具有历史意义或启发性

### 4. 改进网站

欢迎提交 PR 改进网站本身。请注意：

- **保持纯手写**：不引入任何框架、构建工具或包管理器
- **保持像素风**：设计上保持现有的像素 / 复古游戏风格
- **保持幽默**：严肃的事情不严肃地说

## 提交规范

- Issue 标题清晰，中英文均可
- PR 请简要说明改了什么
- 请**手动**提交（开个玩笑，用 AI 辅助写也行，我们不查）

## 行为准则

- 友善交流
- 不贬低使用 AI 的开发者——我们保护手艺，不制造对立
- 尊重每个人的开发方式
