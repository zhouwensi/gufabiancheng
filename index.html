壹 · WHAT

何为古法编程

一种正在消失的、以人类双手和大脑为核心工具的编程方式

古法编程（Artisanal Programming），又称手工编程、人肉编码，是指程序员不借助 AI 代码生成工具，完全依靠自身对编程语言的理解、对文档的阅读、对 bug 的肉眼观察，以及对 Stack Overflow 的虔诚搜索，来完成软件开发全过程的一种传统技艺。

该技艺主要包括但不限于以下实践：

  • ⌨️ 手动逐字输入代码，含标点符号
  • 🔍 肉眼 debug，用 print 语句定位问题
  • 📖 阅读官方文档而非询问 AI
  • 🧠 在大脑中构建程序逻辑
  • ⚙️ 手动配置开发环境与部署流程
  • 🤝 与同事进行人对人的 Code Review

贰 · SKILLS

濒危技艺图鉴

这些曾经是每位程序员的基本功，如今正在成为"非遗"

🖊️

手写 for 循环

濒危程度：极高

从初始化到条件判断到步进，每一个分号都由人类亲手敲入。AI 时代后，此技能退化速度最快。

🐛

肉眼 Debug

濒危程度：极高

在数百行代码中凭直觉和经验找到 bug 的位置。需要对代码有"手感"，讲究望、闻、问、切。

📟

命令行操作

濒危程度：高

不依赖 GUI，纯键盘操作终端。ls、cd、grep、awk、sed 信手拈来，管道符用得行云流水。

📝

Vim/Emacs 编辑

濒危程度：高

在没有智能补全的编辑器中自由编码。退出 Vim 曾是初级测试，如今连进入 Vim 的人都越来越少。

📋

阅读官方文档

濒危程度：极高

直接阅读 RFC、man page、API 文档。不问 AI，不搜博客，从原始资料获取第一手知识。

🔧

手写 Makefile

濒危程度：中

亲手编写编译规则，理解依赖关系和构建顺序。Tab 和空格的区别，是入门第一课。

🌿

手动 Git 操作

濒危程度：高

rebase、cherry-pick、reset --hard，每一步都了然于胸。解决 merge conflict 不靠 AI，靠阅读 diff。

🏗️

手动部署上线

濒危程度：中

SSH 登录服务器，手动拉代码、编译、重启服务、看日志。凌晨三点的发布，是古法程序员的成人礼。

✍️

手写正则表达式

濒危程度：极高

不借助任何工具，凭记忆写出正确的正则。此技能已被 WHO（World Hacker Organization）列为极度濒危。

🖥️

手配 Nginx/Apache

濒危程度：高

反向代理、负载均衡、SSL 证书，全部手写配置文件。每一个 location 块都是匠人的讲究。

🧮

手推算法复杂度

濒危程度：中

在白板上用笔推导时间空间复杂度，画递归树，写状态转移方程。纸和笔，是最古老的 IDE。

💬

写代码注释

濒危程度：极高

"AI 能读懂代码，为什么还要写注释？"——这句话标志着此技艺正式进入消亡倒计时。

叁 · COMPARE

古今对照

两种编程方式的真实对比，各有千秋

场景 🏺 古法 🤖 新法 点评
写一个函数 思考 10 分钟，敲 5 分钟 描述需求 10 秒，AI 生成 1 秒 古法多了思考，也多了理解
Debug 加 print、读日志、冥想 2 小时 把报错贴给 AI，2 秒出答案 古法虽慢，但你真懂了为什么错
学新框架 读文档、写 Demo、踩坑三天 让 AI 直接生成项目脚手架 古法三天，顶新法三个月的理解深度
配置环境 手动装依赖，解决版本冲突半天 "帮我配一下开发环境" → Done 古法练的是系统理解力
写正则表达式 对着文档，测试半小时 描述匹配规则，AI 秒出 古法写完的成就感，无价
Code Review 逐行阅读，写详细评论 AI 自动扫描，生成建议 古法 Review 是一种师徒传承
上线部署 SSH、手动操作、紧张出汗 CI/CD 全自动，推代码即上线 古法凌晨发版的紧张感是一种仪式
遇到不会的 Google → Stack Overflow → 试错 直接问 AI → 复制粘贴 古法练搜索，新法练提问

肆 · LEVELS

修炼等级

古法编程的修行之路，从入门到宗师

🥚

凡人

Level 0 · 尚未入门

写 Hello World 需要问 AI，复制粘贴后不知道为什么能跑。认为"编程就是跟 AI 对话"。

🌱

入门

Level 1 · 初窥门径

能不查资料写完一个 for 循环。知道 = 和 == 的区别。偶尔会手动打开终端输入命令。

⚔️

练气

Level 2 · 能独立存活

能手写完整函数并一次编译通过。会用 git add、git commit、git push 三连。debug 时已经不完全依赖 AI。

🔥

筑基

Level 3 · 基础扎实

能独立阅读报错信息并定位问题。手写 SQL 不会忘了 WHERE 子句。知道什么是 Big O，并且不只是面试时才想起来。

💫

金丹

Level 4 · 独当一面

能不借助 AI 从零搭建一个完整项目。手写正则表达式成功率超过 50%。git rebase -i 运用自如。别人来问你问题，而不是 AI。

🐉

元婴

Level 5 · 高手境界

能在 Vim 中高效编码且不想退出。手写 Makefile 和 Dockerfile 一气呵成。线上故障时，你是最后一道人肉防线。

🏔️

大乘

Level 6 · 近乎传说

能手写编译器或操作系统内核的一部分。看汇编代码就能推断高级语言的逻辑。你不 Google 问题——问题 Google 你。

👑

宗师

Level MAX · 活的传说

你就是文档本身。编程语言因你而修改规范。你的代码无需注释——因为后人会为你的代码写注释。据传全球仅存不超过十位。

传承宣言

我们并不反对 AI。
AI 是好工具，正如电锯是好工具。
但木匠不会因为电锯的存在，就忘记手锯的纹理。

古法编程不是守旧，而是根基。
是你在 AI 给出错误答案时，能识别出来的能力。
是你在网络断开时，依然能写代码的底气。
是你理解"为什么"而不只是"怎么做"的那份深度。

也许有一天，手写代码会像书法一样，
不再是生存技能，而成为一种修行。
但正因如此，它才更值得被记住。

我们记录它，不是因为它即将消亡，
而是因为它值得被铭记。
