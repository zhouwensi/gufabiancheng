# 古法编程

> 纯手工，无添加AI。

在 AI Coding 席卷一切的时代，我们致力于记录和传承**纯手工编程**这门即将失传的人类手艺。

手动写代码、手动 debug、手动部署、手动编译——这些曾经每个程序员的日常，正在成为"非物质文化遗产"。

## 何为古法编程

古法编程，指不借助 AI 辅助工具，由人类程序员凭借自身技艺，亲手完成以下全部或部分工作的编程方式：

- ✍️ **手写代码** — 逐字逐行，从空白文件开始
- 🐛 **手动 Debug** — 肉眼阅读报错信息，用 print 大法定位问题
- 📖 **手翻文档** — 阅读官方文档和源码，而非询问 AI
- 🔧 **手动部署** — 自己写脚本、配环境、改配置文件
- 🧠 **手动设计** — 在白板或纸上画架构图，用脑子想方案

## 古法 vs 新法

```
古法：遇到 bug → 看报错 → 读源码 → 思考 → 加断点 → 定位 → 修复 → 彻底理解
新法：遇到 bug → 复制报错 → 粘贴给 AI → 替换代码 → 能跑就行 → 下一个 bug
```

两条路都能到终点，但走过的路不同。

## 本站技术栈

本站身体力行，采用**纯古法打造**：

- 纯手写 HTML / CSS / JavaScript
- 无框架、无构建工具、无 npm install
- 零 AI 辅助生成页面内容
- 部署于 GitHub Pages

> 好吧最后这条做不到，README 和页面确实问 AI 要的。但至少我们**有这个意识**。

## 传承宣言
